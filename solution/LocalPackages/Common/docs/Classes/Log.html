Log

final public class Log

Logs messages to several LoggerTypes.

Remark

This helps in case it’s needed to log messages not only to the console, but also to other services (crash reports, data base, etc.)

  • setupLoggers(_:)

    Set the collection of loggers to log the messages to.

    Remark

    This can only be called once and is ideally called during the application startup alongside other configuration or setup tasks. Doing this as early as possible will ensure consistency and that all messages get logged.

    Remark

    All other parameters are set by default corresponding the place at the log was called. No messages will be logged if until setupLoggers is called.
